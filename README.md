# Практическая работа №2: Криптографические алгоритмы

## Описание

Полнофункциональная реализация криптографических алгоритмов на C++, включая:
- Модульное возведение в степень (теорема Ферма и бинарный алгоритм)
- Расширенный алгоритм Евклида
- Криптосистема Эль-Гамаля с блочным шифрованием
- Эмуляция атаки "Человек посередине" (MITM)
- Решение линейных диофантовых уравнений через цепные дроби

## Функциональность

### Меню программы

Программа работает через интерактивное меню с шестью основными заданиями:

```
========================================
           Практическая работа №2
========================================
  1 - Модульное возведение в степень (a^x mod p)
  2 - Расширенный алгоритм Евклида (c*d mod m = 1)
  3 - Расширенный алгоритм Евклида (c^-1 mod m = d)
  4 - Шифрование Эль-Гамаля
  5 - Атака MITM на шифр Эль-Гамаля
  6 - Цепные дроби и диофантово уравнение
  0 - Выход
==========================================
```

### Задание 1: Модульное возведение в степень

Реализованы два метода вычисления a^x mod p:
- **Метод Ферма**: использует теорему Ферма (работает только для простых модулей)
- **Бинарный алгоритм**: универсальный и эффективный метод

Программа проверяет условия применимости теоремы и выводит таблицу вычисления.

```
Введите основание a: 2
Введите степень x: 100
Введите модуль p: 101

Вычисляем: 2^100 mod 101

Метод 1 (теорема Ферма): 2^100 mod 101 = 1
Метод 2 (бинарный алгоритм): 2^100 mod 101 = 1
```

### Задание 2-3: Расширенный алгоритм Евклида

Вычисление модульного обратного элемента с выводом таблицы итераций:

```
--- Задание 2: c * d mod m = 1 ---

Введите число m: 26
Введите число c: 5

|      Q|      x|      y|      d|      b|
-----------------------------------------
|      5|      1|      0|     26|      1|
|      1|     -1|      1|      1|      0|

Найденное число d: 21
Проверка условия c * d mod m = 1: 1
```

### Задание 4: Шифрование Эль-Гамаля

Три режима работы:
- **Режим 1**: Шифрование файла
- **Режим 2**: Расшифрование файла
- **Режим 3**: Шифрование текста из консоли

Особенности:
- Автоматическое добавление padding (PKCS#5)
- Блочное шифрование (размер блока: 64 байта)
- Сохранение ключей в бинарный файл `elgamalKey.bin`
- Поддержка произвольных файлов (текст, изображения, архивы)

### Задание 5: Атака MITM

Эмуляция атаки "Человек посередине" в 5 шагов:
1. Алиса генерирует ключи и публикует открытый ключ
2. Ева перехватывает и подменяет ключ Алисы своим
3. Боб шифрует сообщение "ключом Алисы" (фактически ключом Евы)
4. Ева расшифровывает, читает и модифицирует сообщение
5. Алиса расшифровывает подменённое сообщение

```
ШАГ 1: Алиса публикует открытый ключ
[Алиса] Генерация пары ключей...
[Алиса] Открытый ключ сгенерирован:
  p (hex, первые 32 символа): 1c5f82e0a9f3b...
  g (hex): 2
  y (hex, первые 32 символа): 8e47c2d5f1a9...
```

### Задание 6: Цепные дроби и диофантово уравнение

Решение уравнения **1256a + 847b = 119**:

```
Цепная дробь для 1256/847:
[1;2;2;2;4;2]

НОД(1256,847): 119

Частное решение уравнения:
a = 57
b = -85

Проверка:
1256 * 57 + 847 * (-85) = 119

Общее решение:
a = 57 + 7*t
b = -85 - 10*t
```

## Технические детали

### Используемые библиотеки

- **GMP (GNU Multiple Precision Arithmetic)**: для работы с числами до 1024 бит
- **Standard C++ Library**: контейнеры, потоки, работа с файлами

### Параметры криптосистемы

- **Размер простого модуля (p)**: 1024 бита (~309 десятичных цифр)
- **Размер блока**: 64 байта
- **Режим шифрования**: Блочный режим Эль-Гамаля
- **Padding**: PKCS#5 (добавляется до кратности 64 байтам)

### Сложность алгоритмов

| Алгоритм | Сложность | Примечание |
|----------|-----------|-----------|
| Бинарное возведение в степень | O(log n) | где n - значение степени |
| Расширенный алгоритм Евклида | O(log min(a,b)) | линейная по количеству цифр |
| Шифрование блока | O(1) | для фиксированного размера |
| Поиск генератора | O(1) в среднем | с ограниченным числом попыток |

## Решение проблем

### Ошибка: "cannot find -lgmp"

**Решение**: Установите GMP
```bash
# Ubuntu/Debian
sudo apt-get install libgmp-dev

# macOS
brew install gmp
```

### Ошибка при компиляции: "undefined reference to `__gmpz_...`"

**Решение**: Убедитесь, что флаги `-lgmp -lgmpxx` указаны при линковке

### Программа падает при больших числах

**Решение**: Это нормально поведение - используется стандартная глубина рекурсии. Для огромных чисел нужно увеличить лимит стека:
```bash
ulimit -s unlimited
./cryptography_lab
```

### На Windows: программа не запускается

**Решение**: Используйте WSL2 с Linux или установите MinGW:
```bash
# Через WSL2 (рекомендуется)
wsl --install Ubuntu
```

## Тестирование

### Встроенные тесты

Программа содержит встроенные проверки корректности:
- Проверка теоремы Ферма (a^(p-1) ≡ 1 mod p для простого p)
- Проверка расширенного алгоритма Евклида (ax + by = gcd(a,b))
- Проверка шифрования/дешифрования (plaintext == decrypt(encrypt(plaintext)))
- Проверка атаки MITM (модификация данных обнаруживается)
